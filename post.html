Dispelling A Myth: Responsive Images

I recently read Implementing Responsive Design by Tim Kadlec. It was a really well put together book with great case studies and practical advice from a number of industry veterans. However there was a small bit of advice on responsive images which was incorrect:

“It’s important to note that the img element in your HTML cannot have the height and width attributes set. If those values are set, the image will not scale proportionally. For fluid images to work, you need to control the dimensions through CSS alone.”

In code, this would look like:

<img class="responsiveImage" src="funny-cats.jpg" alt="" />

.responsiveImage {
    max-width: 100%;
}

Unfortunately there’s a problem with this recommendation – if you forgo the width and height attributes on your images then the browser doesn’t know how big the images should be until it begins downloading them. This means that the browser will begin to layout your page as soon as content is available and then, once the images begin to download, the browser will have to recalculate the layout again and again to account for the size of each image.

For users this is a jarring experience – it makes your site seem slow, and it can cause the user to get disorientated or lost as the layout of your page changes.

Luckily, you can have responsive images which scale proportionally while also having width and height attributes. Set the max-width property to 100% like normal, and then set the height property to auto:

<img class="responsiveImage" src="funny-cats.jpg" width="640" height="480" alt="" />

.responsiveImage {
    height: auto;
    max-width: 100%;
}

Your images will still scale proportionally, but now the browser will know the image dimensions so it can figure out how to layout the page before downloading the images.
